# Research: OpenEmu/VICE-Core as a native Commodore 64 core candidate

## Summary
OpenEmu/VICE-Core appears to be an old, unfinished OpenEmu core-plugin wrapper around VICE's C64 emulator rather than a current, maintained native Commodore 64 core. It may be useful as historical reference for OpenEmu SDK integration, but it is unlikely to build cleanly for OpenEmu-Silicon without substantial modernization of the Xcode project, OpenEmu plugin glue, vendored VICE source, generated config headers, and Apple Silicon build settings.

## Findings
1. **Project shape: standalone OpenEmu core plugin, not an in-tree OpenEmu-Silicon core.** The repository is a separate OpenEmu-org core repo, with the expected old OpenEmu plugin layout: an Xcode project, plugin Info.plist/resources, Objective-C/Objective-C++ glue, and a vendored or embedded VICE source tree. That means adoption would likely require importing or flattening the project into the OpenEmu-Silicon workspace rather than simply enabling an existing target. [Repo](https://github.com/OpenEmu/VICE-Core) · [Project file](https://github.com/OpenEmu/VICE-Core/blob/master/VICE.xcodeproj/project.pbxproj)
2. **Build target availability is historical, not Silicon-ready.** The upstream repo appears to provide an Xcode target for building a VICE `.oecoreplugin`, but that target predates current OpenEmu-Silicon conventions, Xcode 26, Swift/ObjC modernization, and the fork's flattened-core workflow. It should be treated as a porting source, not as a drop-in target. [Project file](https://github.com/OpenEmu/VICE-Core/blob/master/VICE.xcodeproj/project.pbxproj) · [OpenEmu-Silicon core workflow reference](https://github.com/nickybmon/OpenEmu-Silicon)
3. **Scope appears to be C64/x64 rather than full VICE machine coverage.** The candidate is relevant to issue #542 because VICE is the canonical Commodore emulator family, but the old OpenEmu wrapper appears focused on the C64-side VICE target (`x64`) rather than exposing every VICE machine as separate OpenEmu systems. Verify exact system IDs in `Info.plist` before assuming VIC-20, C128, Plus/4, PET, etc. are covered. [Info.plist](https://github.com/OpenEmu/VICE-Core/blob/master/VICE-Info.plist) · [Repository](https://github.com/OpenEmu/VICE-Core)
4. **Emulator version is apparently very old.** The embedded VICE source should be checked through its version/header/config files, but this repo predates modern VICE releases by many years; current upstream VICE development has moved far beyond the likely version carried here. This increases risk around macOS build fixes, ARM64 assumptions, audio/video APIs, and generated configuration files. [VICE-Core source tree](https://github.com/OpenEmu/VICE-Core/tree/master/vice) · [Upstream VICE project](https://vice-emu.sourceforge.io/)
5. **Last activity appears stale.** The repository's commit history should be treated as dormant OpenEmu-era work rather than an actively maintained core. For decision-making, use the repo commits page as the primary evidence of last activity and compare it with the active upstream VICE release history. [Commit history](https://github.com/OpenEmu/VICE-Core/commits/master/) · [OpenEmu organization](https://github.com/OpenEmu)
6. **Likely dependencies/blockers: generated VICE config, native C/ObjC glue, legacy OpenEmu SDK API, and architecture settings.** VICE traditionally relies on generated `config.h`-style build output and platform feature detection; an Xcode-imported snapshot commonly hardcodes those decisions. Apple Silicon port work would likely need: regenerate or audit config headers for arm64 macOS; remove x86-only build settings; update deprecated OpenEmu SDK calls; ensure the core bundle links only arm64-safe libraries; map keyboard/joystick input through current OpenEmu responder APIs; and verify audio/video timing on M-series Macs. [VICE source tree](https://github.com/OpenEmu/VICE-Core/tree/master/vice) · [VICE build/source docs](https://sourceforge.net/p/vice-emu/code/HEAD/tree/trunk/vice/)
7. **WIP/not-working risk is high.** The repo should not be assumed shippable: it is old, separate from the maintained OpenEmu-Silicon workspace, and likely never reached the same completeness level as shipped OpenEmu cores. Treat it as reference code unless a local build proves the target compiles and the plugin can boot a C64 ROM/game under current OpenEmu-Silicon. [README](https://github.com/OpenEmu/VICE-Core/blob/master/README.md) · [Issues/PRs](https://github.com/OpenEmu/VICE-Core/issues)

## Gaps
Live source inspection should confirm exact values before implementation: final commit date/hash, README wording about WIP status, exact VICE version macro, actual Xcode target names, supported OpenEmu system identifiers, required third-party libraries, and whether the project still contains submodules. Recommended next step: clone `https://github.com/OpenEmu/VICE-Core`, inspect `README.md`, `VICE.xcodeproj/project.pbxproj`, `VICE-Info.plist`, `vice/src/version.*`, and run a local `xcodebuild -list` / arm64 build attempt in an isolated branch or worktree.
